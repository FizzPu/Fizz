Jdk8的下载

    1. 首先要明确的是jre和jdk的区别，jre是java运行时环境，包含了运行java程序的基本环境如jvm, 核心类库。而jdk是给开发者用的，包含了一些其他的工具，如编译器javac, 监视工具jconsle。jdk里面也包含了jre, 因为javac这些工具是需要jre来运行的。
    1. oracle jdk商用是要付费的，不过自己学习用平时用没关系的，当然也可以选择open jdk。这里我选择oracle jdk8。链接https://www.oracle.com/java/technologies/javase/javase8-archive-downloads.html
    1. 通过uname -a 查看机器架构后，选择对应的版本即可

  1. 下载后，将下载的压缩包解压到安装目录即可, 这里我解压到/usr/local/java8目录

    1. 这里我把系统变量配置到用户目录下，而不是root用户, 我使用的是.zshrc终端，所有我编辑.zshrc文件。其他终端用户编辑~/.bashrc文件即可。一般不要配置CLASSPATH，会造成一定的污染，我们常常是自己利用java命令时传入。例如java -cp [classpath] Main。

apache.commons.logging的配置

  • 为啥我想配置这个东西， 原因是我每次新开一个项目，都要为这个项目配置一次日志框架，简直巨恼火。所以这里我为jre添加这个外部依赖，避免每次都去配。这个日志是为所有Java程序日志模块提供的一个接口，编码的时候使用接口，可以根据项目随意切换日志的具体实现。这样我自己写一个日志组件，然后去切换项目中原来的使用即可。

  • 去maven仓库下载apache.commons.loggin的jar包后，放到jre的lib目录下。此目录即使不加入classpath, 也是classpath的一部分。

  • 打开IDEA以后，我们可以在external lib看到这个包的具体信息

  • 在classpath下寻找自己的配置文件commons-logging.properties，如果找到，则使用其中定义的Log实现类：org.apache.commons.logging.Log=org.apache.commons.logging.impl.SimpleLog; //使用commons-logging的SimpleLog
    org.apache.commons.logging.Log=org.apache.commons.logging.impl.Log4JLogger;//选择使用log4j
    org.apache.commons.logging.Log=org.apache.commons.logging.impl.Jdk14Logger; //选择jdk的logger

    如果找不到commons-logging.properties文件，则在查找是否已定义系统环境变量org.apache.commons.logging.Log，找到则使用其定义的Log实现类

    否则查看classpath中是否有Log4j的包，如果发现，则自动使用Log4j作为日志实现类。

    否则使用JDK自身的日志实现类（JDK1.4以后才有日志实现类）。

    否则使用commons-logging自己提供的一个简单的日志实现类SimpleLog。
    可见，commons-logging总是能找到一个日志实现类，并且尽可能找到一个最合适的日志实现类。
    为了简化配置，我们可以不使用commons-logging的配置文件，也不设置commons-logging相关的环境变量，只需将log4j的包放入classpath就可以了，这样就可以完成commons-logging与log4j的结合。如果以后不想使用log4j，只需将log4j的包 从classpath中移除就可以了。
